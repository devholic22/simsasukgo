## 📁 구조

```
docs/
├── yyyy-mm-dd/        # 한국 기준 일자별 폴더

│   │   ├── 01-feature-name.md
│   │   └── README.md
```

## 📖 사용 방법

### 1. 기획 문서 작성 (plan/)

각 기능/버그 수정마다 상세한 기획 문서를 작성합니다.
- 요구사항 정의 (AC)
- 아키텍처/설계
- 예상 시간 및 난이도

### 2. 개발 기록 (develop/)

실제 구현 과정을 기록합니다.
- 구현 내용
- 관련 PR/커밋
- 문제점 및 해결책

### 3. 일일 회고 (retro/)

하루를 마무리하며 전체를 정리합니다.
- 완료된 작업
- 진행 중인 작업
- 내일 계획
